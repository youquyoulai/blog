# 平哥偶记 — Hugo 博客使用教程

> 适用对象：零基础用户
> 博客地址：https://www.pgoj.top
> 管理后台：https://www.pgoj.top/admin/

---

## 一、博客文件夹结构说明

你的博客放在 `e:\blog\` 目录下，以下是每个文件夹的作用：

| 文件夹 / 文件 | 作用说明 |
|---|---|
| `content/posts/` | **文章存放处**，每篇文章是一个 `.md` 文件，文件名格式为 `年-月-日-标题.md` |
| `content/pages/` | 独立页面（如「关于我」等），不是日常文章 |
| `static/admin/` | 管理后台的前端代码（网页界面） |
| `static/img/` | 本地存放的图片（头像、logo 等固定图片） |
| `themes/xlxn/` | 博客主题（外观），一般不用动 |
| `archetypes/` | 新文章的模板，控制新建文章时默认有哪些字段 |
| `public/` | Hugo 生成的静态网页文件，部署时上传的就是这个文件夹 |
| `hugo.toml` | **核心配置文件**，设置博客名称、域名、评论系统等 |
| `wrangler.toml` | Cloudflare 部署配置文件 |
| `package.json` | Node.js 依赖配置（图片处理等工具） |

> 💡 **重点**：你只需要关心 `content/posts/` 和 `hugo.toml`，其他文件夹一般不用动。

## 二、发布文章的两种方式

### 方式一：用管理后台（最简单，推荐）

1. 打开浏览器，访问：**https://www.pgoj.top/admin/**
2. 输入管理密码（Token）登录
3. 点击「写文章」Tab
4. 填写标题、选择分类（观感 / 记录）、上传封面图
5. 在富文本编辑器里写内容
6. 点击「发布」

> ✅ 这种方式最简单，不用碰命令行，文章直接发布上线。

---

### 方式二：本地写文章 + 命令行部署（适合喜欢用编辑器的用户）

#### 第1步：新建一篇文章

在 `e:\blog\content\posts\` 目录下，新建一个文件，文件名格式：

```
年-月-日-标题英文或拼音.md
```

例如：`2026-05-11-my-new-post.md`

#### 第2步：写文章头部信息（Front Matter）

每篇文章开头有一段配置信息，用 `---` 包裹，格式如下：

```yaml
---
title: "文章标题"
date: 2026-05-11
lastmod: 2026-05-11
slug: "my-new-post"
categories: ['guangan']
tags: ["标签1", "标签2"]
cover: "https://img.pgoj.top/202605/xxx.webp"
---
```

| 字段 | 说明 |
|---|---|
| `title` | 文章标题，必填 |
| `date` | 发布日期，格式 `年-月-日` |
| `lastmod` | 最后修改日期 |
| `slug` | 文章网址后缀，如 `my-post`，网址就是 `/archives/my-post.html` |
| `categories` | 分类，`guangan` = 观感，`jilu` = 记录 |
| `tags` | 标签，可多个，用中括号括起来 |
| `cover` | 封面图网址（可选） |

#### 第3步：写文章内容

头部信息写完后，空一行，下面就是文章正文，用普通 Markdown 语法写即可。

#### 第4步：本地预览（可选）

在 `e:\blog\` 目录下打开命令行，运行：

```bash
hugo server -D
```

然后打开浏览器访问 `http://localhost:1313/` 就能看到效果。

按 `Ctrl + C` 停止预览。

#### 第5步：提交并推送到 GitHub（触发自动部署）

```bash
# 进入博客目录
cd e:\blog

# 把新文章加入 Git 跟踪
git add content/posts/

# 提交到本地仓库（引号里写本次更新的说明）
git commit -m "发布新文章"

# 推送到 GitHub（Cloudflare 会自动构建并上线，等1-2分钟即可）
git push
```

> 💡 **说明**：你的博客已配置 GitHub 自动部署，只要 `git push` 推送到 GitHub，Cloudflare Pages 会自动构建并发布，不需要手动部署。

## 三、图片管理

### 图片存在哪里？

图片存在 **Cloudflare R2** 云存储里，通过 `https://img.pgoj.top/` 访问。

在管理后台的「图库」Tab 里，可以看到所有图片按**月份文件夹**分组，例如：
- `202605/` — 2026年5月的图片
- `2026slt/` — 2026年随手拍图片

### 如何在文章里插入图片？

在管理后台写文章时，点击「图库选题」按钮，选择图片，系统会自动插入图片链接。

手动插入格式如下：

```markdown
![图片说明](https://img.pgoj.top/202605/图片名字.webp)
```

## 四、线上有更新时，如何拉回本地

如果你在管理后台（网页）发布了文章，本地电脑上的文件不会自动更新，需要手动拉取。

### 完整步骤

**第1步**：打开命令行（PowerShell 或 CMD），进入博客目录：

```bash
cd e:\blog
```

**第2步**：查看远程有什么更新：

```bash
git fetch
```

**第3步**：拉取最新内容到本地：

```bash
git pull
```

**第4步**：确认拉取成功，查看最新文章：

```bash
ls content/posts/
```

### 常见问题

| 问题 | 解决方法 |
|---|---|
| 提示 `error: Your local changes to the following files would be overwritten by merge` | 先运行 `git stash`，再运行 `git pull`，最后运行 `git stash pop` |
| 提示 `fatal: not a git repository` | 确认当前目录是否正确，应该能看到 `.git` 文件夹 |
| 拉取后文章还在，但图片不见了 | 图片存在云端 R2，不是 Git 管理的，重新打开管理后台即可看到 |

## 八、修改模板（改外观）

模板文件在 `themes/xlxn/layouts/` 目录下，修改后需要重新构建部署才能生效。

### 模板文件结构

| 文件路径 | 作用 |
|---|---|
| `layouts/index.html` | 网站首页 |
| `layouts/posts/single.html` | 文章详情页 |
| `layouts/posts/list.html` | 文章列表页 |
| `layouts/partials/head.html` | 网页头部（引入 CSS/JS） |
| `layouts/partials/header.html` | 顶部导航栏 |
| `layouts/partials/footer.html` | 底部区域 |
| `layouts/_default/single.html` | 通用单页模板（页面/文章共用） |
| `layouts/_default/list.html` | 通用列表模板 |
| `layouts/404.html` | 404 错误页 |

### 修改模板后的推送步骤

```bash
cd e:\blog

# 查看你改了哪些文件
git status

# 把修改加入提交
git add themes/xlxn/layouts/
# 如果还改了 hugo.toml，也要加进去：
git add hugo.toml

# 提交
git commit -m "修改模板：说明改了什么"

# 推送（Cloudflare 自动重新构建上线）
git push
```

> ⚠️ **注意**：修改模板有风险，改错可能导致网站显示异常。建议每次只改一个小地方，改完本地预览（`hugo server -D`）确认没问题再推送。

## 九、更换主题（更换模板）

如果你想换一个全新的博客外观，步骤如下：

### 第1步：下载新主题

到 [Hugo Themes](https://themes.gohugo.io/) 挑选喜欢的主题，下载后解压到 `themes/` 目录下。

```bash
# 例如下载一个新主题
cd e:\blog\themes\
git clone https://github.com/作者/主题名.git
```

### 第2步：修改 hugo.toml

打开 `e:\blog\hugo.toml`，把 `theme = 'xlxn'` 改成新主题的名字：

```toml
theme = '新主题文件夹名'
```

### 第3步：检查并更新 Front Matter（重要！）

不同主题的 **Front Matter 字段可能不一样**。换主题后，原来文章的标题、日期可能显示不正常。

常见问题对照表：

| 问题 | 原因 | 解决方法 |
|---|---|---|
| 文章列表不显示 | 新主题用的 section 名称不同 | 检查主题文档的 `content/` 示例结构 |
| 封面图不显示 | 新主题用 `image` 而不是 `cover` | 批量替换文章里的 `cover:` 为 `image:` |
| 分类/标签消失 | 新主题的 taxonomy 配置不同 | 对比 `hugo.toml` 里的 `[taxonomies]` 配置 |
| 评论区消失 | 新主题没有集成评论系统 | 需要手动在模板里加 Waline/Twikoo 代码 |

### 第4步：本地预览确认

```bash
cd e:\blog
hugo server -D
```

打开 `http://localhost:1313/` 逐个页面检查，确认文章显示正常、样式没问题。

### 第5步：推送上线

```bash
cd e:\blog
git add .
git commit -m "更换主题为 xxx"
git push
```

## 十、注意事项

1. **文章草稿**：在文章头部写 `draft: true`，这篇文章就不会出现在网站上，适合写一半的文章。
2. **删除文章**：直接在 `content/posts/` 里删除对应的 `.md` 文件，然后 `git push` 即可。
3. **修改已发布的文章**：直接编辑对应的 `.md` 文件，修改 `lastmod` 日期，然后 `git push`。
4. **备份**：你的博客已经存储在 GitHub（`youquyoulai/blog`），即使本地文件丢失，也可以从 GitHub 拉回来。
5. **改模板前先备份**：修改 `themes/` 里的文件前，建议先复制一份原文件，改坏了可以恢复。

## 九、修改博客模板（主题定制）

### 模板文件夹结构

模板文件都在 `themes/xlxn/` 目录下，主要文件夹作用如下：

| 文件夹 | 作用 |
|---|---|
| `themes/xlxn/layouts/` | **页面模板**（主页、文章页、分类页等 HTML 布局） |
| `themes/xlxn/assets/` | CSS、JS 等资源文件 |
| `themes/xlxn/static/` | 主题内置的固定文件（图片、字体等） |
| `themes/xlxn/i18n/` | 多语言翻译文件 |

### 常用模板文件说明

| 文件路径 | 控制什么页面 |
|---|---|
| `layouts/index.html` | 博客首页 |
| `layouts/_default/baseof.html` | 所有页面的底层框架 |
| `layouts/_default/single.html` | 单篇文章页面 |
| `layouts/_default/list.html` | 文章列表页（分类、标签列表） |
| `layouts/_default/section.html` | 大分类页（观感 / 记录） |
| `layouts/partials/` | 可复用的部件（侧边栏、底部、评论区等） |
| `layouts/posts/` | 文章相关特殊布局 |

### 修改模板后如何推送上线

修改模板文件后，同样通过 Git 推送到 GitHub 即可：

```bash
cd e:\blog

# 把所有修改加入 Git（包括模板文件）
git add .

# 提交（写清楚这次改了什么）
git commit -m "优化文章页面样式"

# 推送到 GitHub
git push
```

> 💡 **重要**：推送后去 Cloudflare Pages 看一下构建日志，确认没有问题。如果 Hugo 构建报错，说明模板语法有问题，需要检查修改的内容。

### 如何更换主题模板

> ⚠️ **警告**：更换主题是较大的改动，可能导致博客样式、URL 结构、评论系统等出现问题，建议先备份。

#### 完整步骤

**第1步**：备份当前配置

```bash
# 备份 hugo.toml 配置文件
copy e:\blog\hugo.toml e:\blog\hugo.toml.backup
```

**第2步**：下载新主题

```bash
cd e:\blog\themes

# 下载新主题（以 hugo-theme-even 为例）
git clone https://github.com/olOwOlo/hugo-theme-even themes/even
```

**第3步**：修改 `hugo.toml` 中的主题名称

把 `theme = "xlxn"` 改成新主题的文件夹名，例如：

```toml
theme = "even"
```

**第4步**：本地预览测试

```bash
cd e:\blog
hugo server -D
```

打开 `http://localhost:1313/` 查看效果，有问题继续调整配置。

**第5步**：确认没问题后推送上线

```bash
git add .
git commit -m "更换主题为 hugo-theme-even"
git push
```

#### 更换主题可能需要调整的内容

| 内容 | 说明 |
|---|---|
| `hugo.toml` 配置 | 新主题可能有不同的配置项，需要参考主题文档调整 |
| 文章 URL | 不同主题的 URL 结构可能不同，可能导致旧链接失效 |
| 评论系统 | 需要在新主题里重新配置 Twikoo / Waline 评论 |
| 图片 CDN | 图片引用方式是否兼容 |
| 自定义 CSS/JS | 如果有自定义样式，可能需要在新主题里重新添加 |
| 分类/标签页面 | 不同主题的分类展示方式不同 |

#### 快速检查清单

上线前确认以下内容是否正常：

- [ ] 文章能正常打开
- [ ] 首页显示正常
- [ ] 分类页和标签页能访问
- [ ] 评论区能加载
- [ ] 图片能正常显示
- [ ] RSS 订阅正常
